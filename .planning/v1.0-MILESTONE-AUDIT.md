## Cross-Phase Integration Issues

The integration checker found **3 gaps** where Phase 4's new content sources are not fully surfaced in the tool layer:

### INT-01: Search tool source filter missing "model" and "research"

**File:** `src/tools/search.ts:57`
**Issue:** The zod enum for the `source` parameter is `["all", "platform", "code", "api-reference", "blog"]`. Users cannot filter search results to only model or research content. Model/research pages DO appear in unfiltered search (source="all") because the SQL has no filter, but source-scoped queries reject "model"/"research" at input validation.
**Affects:** CONT-01, CONT-02

### INT-02: Search metadata footer ignores model/research staleness

**File:** `src/tools/search.ts:10-42`
**Issue:** `buildMetadataFooter` only knows about doc and blog timestamps. When search results contain `source: "model"` or `source: "research"`, those sources pass through the doc branch and their staleness is checked against the doc crawl timestamp — not the model/research crawl timestamps. A stale model index (not crawled recently) will appear "fresh" if docs were recently crawled.
**Affects:** TRST-01, TRST-02

### INT-03: list_doc_sections tool does not render model/research pages

**File:** `src/tools/list-sections.ts:40-89`
**Issue:** Pages grouped into 4 hardcoded categories (Platform, API Reference, Claude Code, Blog). Model/research pages pass the SQL query but are silently dropped from the display output.
**Affects:** CONT-01, CONT-02
