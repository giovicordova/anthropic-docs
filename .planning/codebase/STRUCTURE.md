## Directory Purposes

**`src/`:**
- Purpose: All application source code
- Contains: 6 TypeScript files, each with a single responsibility
- Key files: `src/index.ts` (entry point), `src/database.ts` (data layer), `src/parser.ts` (doc parsing)

**`tests/`:**
- Purpose: Unit tests for parser, blog-parser, and database modules
- Contains: 3 test files mirroring src/ modules (no test for index.ts)
- Key files: `tests/database.test.ts`, `tests/parser.test.ts`, `tests/blog-parser.test.ts`

**`dist/`:**
- Purpose: Compiled JavaScript output
- Contains: Generated `.js` and `.d.ts` files
- Generated: Yes (by `npm run build` / `tsc`)
- Committed: No (gitignored)

**`docs/plans/`:**
- Purpose: Historical design documents and implementation plans
- Contains: Markdown files with date-prefixed names
- Key files: Design docs for v2 rewrite, blog indexing, MCP improvements

**`.claude/`:**
- Purpose: Claude Code tool configuration
- Contains: Hooks, settings, MCP test logs
- Key files: `.claude/hooks.json`, `.claude/settings.json`

**`.planning/codebase/`:**
- Purpose: GSD codebase analysis documents
- Contains: Architecture, structure, conventions, testing, concerns docs
- Generated: By GSD mapping commands

## Key File Locations

**Entry Points:**
- `src/index.ts`: MCP server main entry point. Initializes DB, registers tools, starts stdio transport.
- `package.json` `"bin"` field: Maps `anthropic-docs-mcp` CLI command to `dist/index.js`

**Configuration:**
- `src/config.ts`: All constants -- URLs, timeouts, thresholds, DB path
- `tsconfig.json`: TypeScript compiler options (ES2022, Node16 modules, strict)
- `package.json`: Dependencies, scripts, project metadata

**Core Logic:**
- `src/parser.ts`: Doc fetching and markdown parsing (`fetchAndParse`, `parsePages`, `splitIntoSections`)
- `src/blog-parser.ts`: Blog fetching and HTML-to-markdown conversion (`fetchSitemapUrls`, `fetchBlogPages`, `htmlToMarkdown`)
- `src/database.ts`: SQLite schema, FTS5 search, page retrieval, generation management
- `src/types.ts`: All shared TypeScript interfaces

**Testing:**
- `tests/parser.test.ts`: Tests for markdown parsing and section splitting
- `tests/blog-parser.test.ts`: Tests for sitemap parsing and HTML conversion
- `tests/database.test.ts`: Tests for DB operations using in-memory SQLite

**Runtime Data (not in repo):**
- `~/.claude/mcp-data/anthropic-docs/docs.db`: SQLite database file (created at runtime)

## Naming Conventions

**Files:**
- `kebab-case.ts`: All source and test files use lowercase kebab-case
- `*.test.ts`: Test files mirror source file names with `.test.ts` suffix
- No barrel files (`index.ts` in `src/` is the entry point, not a re-export barrel)

**Directories:**
- `lowercase`: All directories use lowercase names
- `kebab-case` for multi-word directories (e.g., `mcp-test-logs`)

**Imports:**
- Use `.js` extensions in import paths (required by Node16 ESM resolution): `import { foo } from "./bar.js"`
- Type-only imports use `import type`: `import type { ParsedPage } from "./types.js"`

## Where to Add New Code

**New Data Source (e.g., new doc provider):**
- Create parser: `src/{source-name}-parser.ts` (follow `src/blog-parser.ts` pattern)
- Add source type: extend `DocSource` union in `src/types.ts`
- Add config constants: `src/config.ts`
- Add crawl orchestration: `src/index.ts` (new `startXCrawl()` + `checkAndCrawlX()` functions)
- Add tests: `tests/{source-name}-parser.test.ts`

**New MCP Tool:**
- Register in `src/index.ts` using `server.registerTool()` pattern (see lines 179-457 for examples)
- Add any new query functions to `src/database.ts`
- Add any new prepared statements to `Statements` interface in `src/types.ts`

**New Database Query:**
- Add prepared statement to `Statements` interface in `src/types.ts`
- Add statement preparation in `prepareStatements()` in `src/database.ts`
- Add exported query function in `src/database.ts`
- Add tests in `tests/database.test.ts`

**New Shared Type:**
- Add to `src/types.ts` (all interfaces live here)

**New Configuration Constant:**
- Add to `src/config.ts` (all constants live here)

**Utilities:**
- No separate utils directory exists. Small helpers are defined as module-private functions in the file that uses them (e.g., `preprocessQuery` in `src/database.ts`, `fetchWithTimeout` in `src/parser.ts`).
- If a utility is needed by multiple modules, add it to a new `src/utils.ts` file.
